# Wire
选定wire作为后端各个组件的依赖注入代码生成工具。
## 为什么选择它
由于习惯了JAVA中spring 依赖注入控制反转，同时在桌面应用中后续可能会存在十分复杂的业务逻辑各个功能/业务之间盘根错节使用手动管理的方式十分容易出错在go的多个可选方案中最终敲定了Wire(由于没有使用的反射运行速度几乎没有影响)。
## 如何开始
1. 请在wire.go中将各个*提供者*的初始化方法维护在其中即可。维护好后需要手动在项目主目录下执行以下控制台命令来生成依赖注入的代码。
2. 若需要知道更多我建议您翻阅Wire的官方文档或浏览社区资料。
```shell
 wire .\cmd\wire.go
```
